Products

The products page shows a paginated list of all products with a search form. The filter form will fire when the user changes an input, but the search function makes use of the RxJs Debounce function to wait for the user to stop making changes - This means the api call to the server is only attempted when the user finishes typing for 1 second rather than after each key press, reducing the strain on the rest api.
When searching for products and navigating away from the page and returning again, it will remember your search criteria.

Read more on RxJs here: https://angular.io/guide/rx-library

Code Files: Github

Product

The product detail page has 3 tabs showing the edit form for the product, the stock holding for each store and all the orders for that particular product.

The product edit form makes use of Angular Reactive Forms, https://angular.io/guide/reactive-forms.
If you try and change the model year to before 2000 or after 2021 you will get an invalid form error and the submit button will be disbaled until the entire form is valid.

The data for the product is loaded BEFORE the route starts. Sometimes this is done so the user doesn't sit on an empty page waiting for data to load.

Orders

Orders page shows a paginated list of all orders, ordered by order date. The filter form uses the template-driven forms: https://angular.io/guide/forms
The date range form field is using ng-bootstrap, a third party module built by bootstrap for angular.

Code File: Github

Order

An order detail page is protected using a route guard. Meaning you will need to login before you can view this page. Upon logging in you will be redirected the requested page.

Create an Order

This is the most complicated page yet, it makes use of Reactive Forms, using validators, and cross field validation. You can dynamically add products to the form and calculations are made each time you modify a product.

The page also makes use of a 'CanDeactivate' guard, which in this case, if you start an order and try to navigate away from the page, you will recieve a dialog to confirm you want to navigate away and lose changes.

Customers

Customers page shows a paginated list of all customers.

Customer

When navigating to a customer edit/detail page, a dropdown list will appear in the header showing your 5 most recently viewed customers.
This could be helpful in a situation where you are dealing with multiple customers at the same time.
The dropdown list is making use of a Subject within the RxJs toolset: https://rxjs-dev.firebaseapp.com/guide/subject meaning that multiple components can listen to an output as an observable and watch for changes.

Other

HTTP Interceptors

The app using a HTTP Interceptor: https://angular.io/api/common/http/HttpInterceptor which intercepts all http requests and responses and will show a loading overlay if a httprequest in progress.
There are many things you can do with an interceptor, for example you could cache data client side so each time the user requests the same data, we can just get it from the cache to save resource and time.
You could also add headers to each request to a specific URL.

Auth Guard

All pages other than the Home page are protected, so it should redirect the user to login page if they are not authenticated. Some pages will require ADMIN privelidges.
